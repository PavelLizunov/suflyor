## Q1 — Озвучка: цифры и прочее (самое заметное; локально проверяемо)

**Диагноз (подтверждён):** в `tts.rs` есть markdown→speech-очистка
(`clean_for_speech`), но **числовой нормализации нет вообще** — Piper-модели
(Irina/Ruslan) читают «123», «45%», «14:30», «v0.33.0» по-цифрово/криво. SAPI-
фоллбэк числа читает сам — портить его не надо.

**Фиксы:**
- Q1.1 Новый модуль `overlay-backend/src/tts_normalize.rs`: целые числа →
  русские слова (именительный падеж, до триллионов; отриц.), «45 %»/«45%» →
  «сорок пять процентов», время «14:30», диапазоны «3–5» → «от трёх до пяти»
  (упрощённо «три-пять»), версии «v0.33.0» → «вэ ноль тридцать три ноль»,
  десятичные «3.5» → «три и пять». ~15 юнит-тестов.
- Q1.2 Применять ТОЛЬКО в neural-ветке `speak()` (SAPI не трогаем).
- Q1.3 Приёмка: HTML-чеклист из 10 контрольных фраз (числа, %, время, версия,
  смешанный RU/EN) — слушает владелец/тестер.

**Архитектурное решение:** нормализация живёт в backend ПЕРЕД отправкой в
sidecar; `suflyor-tts.exe` остаётся тупым синтезатором (SPEAK text) — не
раздуваем сайдкар, тестируем нормализатор юнитами без аудио.
**Не делаем (YAGNI):** склонение числительных по падежам контекста, полный
ITN — именительного падежа достаточно, чтобы «123» перестало быть мусором.

## Q2 — Диаризация: качество определения говорящих

**Диагноз (подтверждён):** pyannote-segmentation + WeSpeaker-embeddings,
`threshold 0.5`, `min_duration_on 0.3`, ручное число спикеров (дефолт 2);
**пост-обработки сегментов нет** — короткие осколки и дребезг «А-Б-А» на
VoIP-звуке размывают атрибуцию.

**Фиксы (по слоям, дешёвое сначала):**
- Q2.1 Пост-склейка в `diar.rs`: соседние сегменты одного спикера с зазором
  <0.6с сливать; осколки <0.4с приклеивать к соседу/отбрасывать. Юнит-тесты
  на синтетических сегментах.
- Q2.2 A/B embedding-модели: 3D-Speaker ERes2Net (тот же sherpa-релиз) против
  WeSpeaker на 2–3 реальных созвонах владельца (offline, режем на сегменты,
  сравниваем атрибуцию). Если стабильно лучше — новый SHA-pinned asset в
  `diar_install.rs`.
- Q2.3 (бэклог Ф3/D из goal-v0.30-plan) авто-подбор числа спикеров: sweep
  N=2..6 + отсев вырожденных кластеров (elbow). Только после Q2.1/Q2.2.

**Приёмка:** тестер на реальном созвоне размечает 20 реплик — % правильной
атрибуции до/после (числа пишем в retest-HTML).
**Архитектурное решение:** остаёмся на sherpa в сайдкаре (NO new crate);
качество добираем пост-обработкой и моделью, не сменой стека.

## Q3 — SVG-иконки: кривые и повторы

**Диагноз (подтверждён фактами):** 43 иконки; 42 на единой сетке
`viewBox 16×16, stroke 1.6`, **один выброс — `star.svg` (24×24, stroke 2)** —
визуально «кривая/чужая». Побайтовых дублей нет, но семантические повторы
есть: `speaker.svg` ×8 мест (озвучка + вкладка Аудио + бар ×3 + транскрипт),
`play.svg` ×7, `x.svg` ×6 — одна картинка на разные смыслы.

**Фиксы:**
- Q3.1 `star.svg` перерисовать на сетку 16/1.6 (сразу).
- Q3.2 Галерея-HTML всех 43 иконок (сгенерю) — владелец помечает «кривые»;
  помеченные перерисовываю в общем стиле.
- Q3.3 Развести повторы: озвучке — своя voice-иконка, вкладке Аудио — своя,
  плееру и «старту сессии» — разные play-варианты (3–5 новых иконок 16/1.6).
- Q3.4 **`tests/icon_guard.rs` в гейт** (по образцу i18n_guard): каждый
  `assets/icons/*.svg` обязан быть `viewBox 0 0 16 16` + `stroke-width 1.6` —
  выбросы больше не проходят.

**Архитектурное решение:** конвенция задокументирована в
`assets/icons/README.md` (сетка, stroke, caps) + guard-тест; новые иконки —
только по конвенции.

## Q4 — Ровность интерфейса

**Диагноз:** точечные неровности (spacing-разнобой 6/8/10/12px по карточкам
Настроек; прецеденты уже чинились: футер плеера, оверфлоу Диагностики).
Конкретный список мест — нужен от глаз владельца/тестера.

**Фиксы:**
- Q4.1 Снимаю галерею скриншотов всех вкладок Настроек + бар + тайл + архив
  (CopyFromScreen по HWND — надёжный канал) → HTML-галерея, владелец краснит
  неровное; итерация фиксов по списку.
- Q4.2 Spacing-токены в `theme.slint` (4/8/12/16) и проход по
  `settings_panel/tile/overlay_bar` — единые отступы вместо магических чисел.
- Q4.3 Конвенция в `docs/design-system.md` (отступы, размеры кнопок, подписи
  полей) — чтобы новые панели не разъезжались.

## Порядок исполнения (предлагаемый)

1. **Q1** (TTS-нормализация) — самый заметный дефект, самодостаточный.
2. **Q2.1** (склейка сегментов) + **Q3.1/Q3.4** (star + icon-guard) — дешёвые.
3. **Q3.2 галерея иконок + Q4.1 галерея скриншотов** — отдаю владельцу на
   разметку (параллельно с п.1–2), фиксы по разметке.
4. **Q2.2** A/B embedding-модели (нужны реальные записи).
5. **Q5.1** прослушка WSOLA → **Q5.2** Slint 1.17 → Q2.3 авто-N → Q5.3.

Каждый пункт — отдельный коммит с гейтом; релиз-пакеты копим (после v0.34.0).
