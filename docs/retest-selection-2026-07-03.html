Суфлёр — ретест: выделение текста + фикс наслоения

Сборка от 2026-07-03 (установлена в %LOCALAPPDATA%\suflyor-slint\ — запусти обычным ярлыком). Заполни статусы + коммент → внизу «Экспорт результатов».

Что в этой сборке

  • Фикс наслоения — блоки (заголовки/код/таблицы/абзацы) теперь считают высоту от текста, не должны налезать друг на друга. Это главное — я это упустил в прошлый раз.
  • Выделение мышью в тайлах / сводках / архиве и в стенограмме + правый клик «Копировать выделенное» / «Выделенное в память».

Чего пока НЕТ (не баг — в работе)

  • Сквозное выделение через несколько блоков («выделить сразу много»). Сейчас выделение — в пределах одного блока. Отдельный режим «Выделить текст» на весь ответ — следующим шагом (Option A). Если попробуешь тащить через блоки и не выйдет — это ожидаемо, отметь как знаешь.

A. Наслоение / высота блоков — ГЛАВНОЕ

Заголовки в сводке не наслаиваются
Открой сводку с разделами (архив F7 → «Просмотреть» сессию со сводкой, или свежую сводку созвона). Разделы вида «Участники», «Обсуждение», «Решения», «Задачи».
Ожидается: каждый заголовок на своей строке, буллеты под ним — не поверх заголовка, текст нигде не наезжает друг на друга.
Код-блок на своей высоте
Открой тайл с блоком кода (ответ ИИ с ```-блоком).
Ожидается: код в рамке занимает свою высоту, следующий блок под ним, не внахлёст.
Таблица на своей высоте
Если попадётся ответ с таблицей — проверь её.
Ожидается: таблица занимает свою высоту, не налезает. (нет таблицы под рукой → «Пропустить»)
Длинный многострочный абзац/буллет
Найди длинный пункт, который переносится на 2–3 строки (сузь тайл, если нужно).
Ожидается: строка растёт под текст, следующий пункт начинается ПОСЛЕ, без наложения.
Обе темы
Переключи оформление (тёмное/светлое) в Настройках и глянь те же блоки.
Ожидается: наслоения нет в обеих темах.

B. Выделение в тайлах / сводках / архиве

Тащишь мышью — выделяется
Зажми и протащи мышь по тексту абзаца/буллета/кода.
Ожидается: текст подсвечивается (акцентный фон).
Двойной / тройной клик
Двойной клик по слову; тройной — по строке/абзацу.
Ожидается: двойной = слово, тройной = абзац.
Ctrl+C копирует выделенное (в т.ч. рус. раскладка)
Выдели фрагмент → Ctrl+C на РУССКОЙ раскладке (физич. клавиша C) → вставь куда-нибудь.
Ожидается: вставился именно выделенный фрагмент.
Правый клик → «Копировать выделенное»
Выдели → правый клик → «Копировать выделенное» → вставь.
Ожидается: скопирован выделенный фрагмент.
Правый клик → «Выделенное в память»
Выдели → правый клик → «Выделенное в память». Внизу тайла появится поле с этим текстом → поправь при желании → «В память».
Ожидается: фрагмент попал в Настройки → Память (проверь там).

C. Выделение в стенограмме

Текст реплики выделяется
Архив F7 → 📄 на сессии со стенограммой → протащи мышь по тексту реплики.
Ожидается: текст выделяется.
Перемотка (seek) — по таймкоду/имени
Клик по таймкоду или имени спикера слева (не по тексту реплики).
Ожидается: перемотка на этот момент работает. (Клик по самому тексту теперь выделяет — так задумано.)
Правый клик → «Выделенное в память» (стенограмма)
Выдели часть реплики → правый клик → «Выделенное в память» → редактор → сохрани.
Ожидается: фрагмент в Настройки → Память.

D. Общее

Ничего не сломалось из старого
Быстро прогони привычное: скролл в тайле, перетаскивание тайла за шапку, кнопки 📋-копия и ⭐ на блоках, «Продолжить диалог».
Ожидается: всё как раньше.
