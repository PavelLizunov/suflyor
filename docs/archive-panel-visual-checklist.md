# Архив-панель — чек-лист живой проверки (перед «релизь»)

Все 4 ТЗ собраны, прошли гейт (fmt + clippy -D + test ×3 + i18n_guard), cargo-deny
(rodio чист) и состязательный ревью-агент (0 critical / 0 important на каждый
инкремент). Boot-smoke: приложение стартует без падений со всеми изменениями.

ОСТАЛОСЬ ТОЛЬКО ЧЕЛОВЕЧЕСКОЕ (по сквозным требованиям ТЗ): «визуальная проверка
глазами» + «живая проверка на слух» (плеер) + явное «релизь». Этот файл — структура
для прохода. Запуск: `cargo run --bin overlay-host` (из `slint-experiment`), затем
**F7** (архив) → **📄** в строке сессии С ЗАПИСЬЮ.

## ТЗ1 — Стенограмма (окно `transcript.slint`, открывается по 📄 в строке архива)

- [ ] Окно открывается, заголовок «Стенограмма» (RU, НЕ английский fallback).
- [ ] Хронология строк: `[mm:ss]` · **Система/Микрофон** (RU) · текст; таймкоды и
      метки совпадают с тем, что показывает markdown-вид сессии.
- [ ] Длинные реплики переносятся по словам (не обрезаются).
- [ ] Старая сессия БЕЗ строк → «Транскрипт не сохранён», без падения, без пустой
      колонки времени.
- [ ] «Копировать всё» → буфер содержит «Спикер: текст» по строкам; кнопка мигает
      «Скопировано ✓» ~1.5с.
- [ ] Галка «с таймкодами [mm:ss]» ON → копия с префиксом `[mm:ss] `.
- [ ] Чекбоксы строк + «выделить все»: отметить часть → «Копировать выбранное» →
      в буфере ТОЛЬКО выбранные, в хронологии. Пусто-выбрано → no-op.
- [ ] Esc закрывает окно; перетаскивание за заголовок двигает окно.
- [ ] Окно НЕ светится в скрин-шеринге при включённом стелсе (WDA) — и при
      переключении стелса ON уже-открытое окно перестаёт быть видимым в захвате.

## ТЗ2а — Удаление (🗑 в строке архива)

- [ ] 🗑 → нативный диалог подтверждения; отмена → ничего не удаляется.
- [ ] Подтверждение → сессия исчезает из списка сразу и НЕ находится поиском.
- [ ] Проверить на диске: `.jsonl`, `recordings/<id>/`, `conspects/<id>.json` и
      строка каталога (SQLite-каскад) стёрты.
- [ ] Активную (текущую) сессию удалить нельзя; повтор удаления идемпотентен.

## ТЗ2б — Плеер (мини-плеер внизу окна стенограммы) — ЖИВАЯ ПРОВЕРКА НА СЛУХ

- [ ] **Клик по строке** → микс (mic+system) играет С МОМЕНТА этой строки (на слух!).
- [ ] **▶/⏸** play/pause работает; повторный клик по строке — seek туда и играет.
- [ ] **Seek-bar**: клик в произвольную точку → перескок туда (на слух); полоса
      заполнения движется при воспроизведении.
- [ ] **Время** «mm:ss / mm:ss» идёт и совпадает с реальной позицией.
- [ ] **Подсветка активной строки** следует за воспроизведением.
- [ ] Дослушать до конца → следующий ▶ играет С НАЧАЛА (не с последней точки).
- [ ] Сессия БЕЗ записи → плеера нет, показано «Аудио не сохранено».
- [ ] Закрыть окно во время игры → звук останавливается (плеер сброшен).

## ТЗ3 — Ретро-резюме (↻ Summary в строке архива)

- [ ] У сессии БЕЗ резюме И без аудио кнопка ↻ активна → строит резюме из
      транскрипта (caталог → jsonl), прогресс + стоимость, язык = response_language.
- [ ] Ошибка → «Повторить», сессия не портится; нет данных → неактивна.

## Сквозное (release-protocol)

- [ ] Все видимые строки на RU (нет английского fallback) — особенно новые:
      Стенограмма, Транскрипт не сохранён, Копировать всё/выбранное, с таймкодами,
      выделить все, Аудио не сохранено.
- [ ] Кнопки/иконки логичны (нет эмодзи там, где должен быть SVG; ▶/⏸ — символы,
      это осознанно для компактного плеера — заменить на SVG, если режет глаз).
- [ ] Темы: проверить хотя бы Light Frost (#3) и одну тёмную — текст кнопок на
      акценте читаем (used `Theme.on-accent`), подсветка строки видна.

## NB

- ArchiveWindow сам имеет ту же пред-существующую дыру стелс-тогла (не в
  WindowRegistry) — заведено ОТДЕЛЬНОЙ фоновой задачей, вне объёма этих ТЗ.
- Эмодзи ▶/⏸ в кнопке плеера — единственное место, где символ вместо SVG (плеер
  компактный); если по гайдлайну нужен SVG — это правка одной строки в transcript.slint.
