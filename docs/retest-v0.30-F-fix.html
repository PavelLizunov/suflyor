Гард переопределения говорящих — повторная проверка после фикса

Что чиним. В прошлый раз F1 не срабатывал («не спрашивает»). Причина (fable нашёл вживую): переименование сохранялось только по Enter — если ты печатал имя и не жал Enter (или кликал «Определить заново», уводя фокус), имя оставалось видимым, но в базу не попадало → гард видел «имён нет» и молча перезапускал. Теперь имя сохраняется по мере ввода (каждый символ), так что гард срабатывает даже без Enter.

Как запустить. Я обновил установленный билд + перезапустил приложение. Стенограмма: F7 → сессия с записью → 💬 → «Определить говорящих».

Только F — G/H/V ты уже принял, их не перепроверяем.

🗣 Гард переопределения

F1 (главное)Печатаешь имя БЕЗ Enter → карточка всё равно появляется
Определи говорящих → в поле имени впиши имя и НЕ жми Enter → сразу жми «Определить заново». Должна появиться карточка «Определить говорящих заново? — Заданные имена будут сброшены — Отмена / Определить заново». (Это тот самый сценарий, что падал.)
F1b. С Enter — тоже появляется
То же, но с Enter после ввода имени → карточка появляется. (Должно работать в обоих случаях.)
F4. «Определить заново» в карточке — переопределяет
В карточке нажми «Определить заново» → запускается определение (имена сбрасываются — но уже осознанно, ты подтвердил). «Отмена» → имена целы, ничего не запускается.
F3. Esc и клик по фону закрывают карточку
Когда карточка открыта: Esc и клик по затемнённому фону закрывают именно карточку (не окно). Имена при этом целы.
F2. Без имён — запускается сразу, без карточки
Свежий результат без заданных имён: повторное «Определить» запускается сразу (терять нечего).
