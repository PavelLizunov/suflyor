suflyor v0.26.0 — ретест

3 ТЗ тестера (2026-07-02): код-блоки у локальной модели · копия блока · память (редактирование + захват). Короткий чек-лист по изменённому.
ВАЖНО: тестируй УСТАНОВЛЕННЫЙ билд (setup.exe /S → ярлык «suflyor (Slint)»), версия = 0.26.0. Чипы RUEN = проверить в обоих языках интерфейса; LOCAL = на локальной модели.
ТЗ-B — локальная модель оформляет код как облако
B — код/команды от ЛОКАЛЬНОЙ модели = оформленный блокfeature
LOCALШаги: переключись на ЛОКАЛЬНУЮ модель. Задай вопрос, ответ на который содержит МНОГОСТРОЧНУЮ команду / SQL / конфиг (напр. «пример docker-compose для nginx», «SQL: выбери пользователей с JOIN», «bash-скрипт бэкапа»). Посмотри ответ в авто-тайле (по детекту вопроса), в ответе на F9, и в «Продолжить диалог».
Ожидание: код выводится ОФОРМЛЕННЫМ блоком (моноширинный шрифт, тёмный фон, ярлык языка сверху) — так же, как у облачной модели. НЕ инлайном с видимыми сырыми `backticks`.
⚠ Это правка ПРОМПТА: слабая локальная модель может иногда не послушаться — если инлайн-код всё же проскакивает, отметь «Не прошло» и приложи пример вопроса/ответа.
B — короткая команда в тексте остаётся инлайн (не раздувается в блок)feature
LOCALШаги: задай вопрос, где в предложении упомянута ОДНА короткая команда/имя (напр. «чем смотреть логи? — journalctl»).
Ожидание: короткая команда в строке — инлайн, НЕ вынесена в отдельный большой блок. (Блок — только для многострочного.)
ТЗ-C — «Копировать» у каждого код-блока
C — своя кнопка «Копировать» у каждого код-блокаfeature
LOCALШаги: получи ответ с код-блоком (или с НЕСКОЛЬКИМИ). Наведи на код-блок → в его углу иконка копирования. Нажми → вставь (Ctrl+V) в блокнот. Повтори на облачной модели.
Ожидание: в буфер попадает ТОЛЬКО код этого блока — чистый, без ``` и без окружающего текста. Иконка на ~1.5с меняется на галочку. Одинаково local/cloud.
C — общая «Копировать» (весь ответ) не сломаласьfeature
Шаги: у тайла нажми общую кнопку копирования (в шапке тайла) → вставь.
Ожидание: копируется весь ответ, как раньше. Две кнопки не конфликтуют.
ТЗ-A1 — редактирование одобренной памяти
A1 — «Редактировать» у одобренного фактаfeature
RUENШаги: Настройки → «Память» → у любого факта в «Одобренной памяти» нажми «Редактировать» → измени текст → «Сохранить». Затем: открой другой на «Редактировать» и нажми «Отмена». Закрой и снова открой приложение, проверь Память.
Ожидание: правка сохраняется и видна сразу; «Отмена» ничего не меняет; после перезапуска правка на месте. Кнопки «Редактировать» и «Удалить» есть у каждого факта.
ТЗ-A2 — «⭐ В память» из ответа / стенограммы
A2 — ⭐ добавить фрагмент ответа/саммари в памятьfeature
RUENШаги: в ответе-тайле (и в 📝 Саммари созвона) у абзаца / буллета / код-блока нажми ⭐ → появится поле с текстом → подправь/сократи → «Сохранить». Открой Настройки → «Память».
Ожидание: ⭐ рисуется как звезда (НЕ квадрат); поле предзаполнено текстом блока; после «Сохранить» факт появляется в «Одобренной памяти». «Отмена» не добавляет.
⚠ После «Сохранить» поле просто закрывается (тоста «сохранено» пока нет — проверяем факт в Настройки→Память). Отметь, если это сбивает с толку.
A2 — ⭐ добавить строку стенограммы в памятьfeature
RUШаги: Архив → открой стенограмму сессии → у строки нажми ⭐ → внизу окна появится поле → подправь → «Сохранить». Проверь Настройки → «Память».
Ожидание: клик по ⭐ на строке НЕ запускает воспроизведение (только захват); поле внизу с текстом строки; факт попадает в «Одобренную память».
A2 — добавленный факт подмешивается в ответы / «Продолжить диалог»feature
Шаги: добавь в память факт (напр. «мой проект называется X, стек Y»). Затем задай вопрос по теме или используй «Продолжить диалог» так, чтобы факт был релевантен.
Ожидание: ассистент учитывает добавленный факт (как и раньше для авто-извлечённой памяти). Авто-извлечение продолжает работать параллельно.
Побочное (в том же релизе)
Очистка «Память»/«Очередь» обновляет список СРАЗУfix
Шаги: Настройки → Диагностика → «Очистить очередь предложений» (2 клика). НЕ закрывая Настройки, перейди на вкладку «Память».
Ожидание: список очищенных записей пуст СРАЗУ (не нужно закрывать и снова открывать Настройки).
