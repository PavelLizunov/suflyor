suflyor × Hermes — приёмка

Мост (агент читает suflyor) + API (suflyor дергает агента для профиля)

Что это. Двусторонний локальный мост между suflyor и твоим Hermes-агентом. Всё localhost-only и по умолчанию выключено.

① Мост (Hermes → suflyor): агент читает созвоны/память/профили и может предлагать факты в память (с твоим одобрением) и создавать профили.

② API (suflyor → Hermes): кнопка «Подготовить профиль» отдаёт вводную агенту (он ресёрчит) и пишет активный профиль. Живые ответы это НЕ трогает.

Разовая настройка перед тестом (всё из приложения, никаких скриптов):
  1. Установи свежий билд из релиза: suflyor-slint-setup.exe (двойной клик, дойди до конца), затем запусти suflyor из меню Пуск. Проверяем именно УСТАНОВЛЕННУЮ версию.
  2. В suflyor: Настройки → Hermes → включи «Мост для Hermes» (токен появится сам). Поле «Хост привязки» оставь 127.0.0.1 — твой Hermes на этой же машине.
  3. Нажми «Установить плагин в Hermes» → статус «готово: плагин установлен и включён…». Кнопка сама кладёт файлы плагина, прописывает токен и включает его в Hermes.
  4. Перезапусти Hermes.
  5. Для ② — включи Hermes API-сервер: в конфиге Hermes (%LOCALAPPDATA%\hermes\config.yaml) поставь platforms.api_server.enabled: true + extra.key: <свой секрет>, перезапусти Hermes, этот же ключ вставь в suflyor → Настройки → Hermes.

① Агент читает suflyor

read«Что было на последнем созвоне?»
Спроси Hermes в чате. Он вызывает suflyor_recent_sessions + suflyor_get_summary/transcript и пересказывает реальный созвон.
＋ скриншот
readПоиск + память
«Найди во всех созвонах, где обсуждали X» (→ suflyor_search). «Что в моей памяти про людей?» (→ suflyor_get_memory).
writeПредложение в память идёт В ОЧЕРЕДЬ (не напрямую)
Попроси Hermes: «предложи в память suflyor, что …» (→ suflyor_suggest_memory). В suflyor это должно появиться в очереди предложений раздела «Память» — на твоё одобрение, НЕ сразу в одобренной.
＋ скриншот

② suflyor дергает агента (профиль)

P1«Проверить» связь с Hermes API
Настройки → Hermes → вставь URL http://127.0.0.1:8642/v1 + ключ (Сохрани) → «Проверить» → [ok] ….
＋ скриншот
write«Подготовить профиль (Hermes)»
Впиши вводную (напр. «Собес в Acme, роль Senior SRE») → «Подготовить профиль». Статус: «Hermes готовит… (долго)» → через время «готово: профиль «…» создан и активен». Проверь в Профилях — новый активный профиль со справкой.
＋ скриншот
readАгент создаёт/меняет профиль обратно
Из чата Hermes: «создай в suflyor профиль для собеса в компанию X и сделай активным» (→ suflyor_set_profile). В suflyor профиль появляется/обновляется + активен.

🔒 Безопасность / регресс

S1Без токена/выключенный мост — молчит
Выключи мост → запрос из Hermes должен вернуть «мост недоступен». (Опц.) Проверь, что мост НЕ отвечает на IP машины в LAN, только на 127.0.0.1.
S2Остальное suflyor не сломалось
Живые ответы (F9), скрины (F8), стенограмма, память — работают как раньше; вкладка Hermes не мешает другим настройкам.
