# suflyor — руководство пользователя

Полное руководство: от первого запуска до ежедневной работы и решения проблем.
Коротко про установку — в [`README.md`](../README.md); здесь — подробно и по шагам.

> **TL;DR для нетерпеливых:** установить → вписать ключи (или поставить локальный
> AI) → запустить сессию (захват звука включается сам) → говорить/слушать, тайлы
> с ответами появляются сами или по **F9**. Перед демонстрацией экрана включить
> **🛡 Скрытность**.

---

## 1. Что это и как работает

suflyor — это плавающий поверх всех окон ассистент для технических
собеседований и встреч под Windows. Он:

1. **слушает звук** — ваш микрофон и/или звук собеседника (системный звук);
2. **расшифровывает речь** (STT) — в облаке (Groq Whisper) или локально
   (GigaAM-v3 / Whisper);
3. **спрашивает LLM** — в облаке (Claude через мост) или локально (llama.cpp);
4. **показывает ответ** в маленьком окошке («тайле») рядом с окном встречи;
5. **прячется от захвата экрана** — при включённой скрытности ни бар, ни тайлы
   не попадают в Teams / Meet / Zoom / OBS / Print Screen.

Всё нативное (Rust + Slint), без браузера. Один пользователь, без телеметрии.

## 2. Установка

1. Скачать **`suflyor-slint-setup.exe`** из
   [Releases](https://github.com/PavelLizunov/suflyor/releases).
2. Запустить. Бинарник не подписан → Windows SmartScreen может предупредить:
   нажмите **«Подробнее» → «Выполнить в любом случае»**.
3. Ставится в `%LOCALAPPDATA%\suflyor-slint\` — **без прав администратора**,
   ярлык появляется в меню Пуск (и на рабочем столе).

Обновления: **Настройки → 🆙 Обновления → 🔍 Проверить обновления** — покажет
заметки к релизу и откроет страницу GitHub для скачивания. Автоустановки нет
(нет подписи — это осознанно). Настройки переживают обновление.

## 3. Первый запуск — что настроить

После установки откройте **⚙ Настройки**. Есть два пути: облако или локально.

### Вариант A — облако (быстрее всего начать)

1. **🧠 AI (мост):** впишите bearer-токен, базовый URL и модель
   (полный ID Claude, напр. `claude-haiku-4-5`). Нажмите **«Проверить
   подключение к AI мосту»** — должно стать «ok».
2. **🗣 STT:** впишите ключ **Groq** (для распознавания речи). Нажмите
   **«Проверить подключение»**.

### Вариант B — локально (ничего не уходит наружу)

Настройки → **AI** → кнопка **«Установить локальный AI»**. Она сама скачает
llama.cpp + whisper.cpp + модели, поднимет локальные серверы, выберет сборку под
видеокарту (CUDA, если есть NVIDIA) и пропишет всё в настройки. Прогресс и режим
GPU/CPU видно прямо в окне. Подробности и ручной скрипт — в
[`README.md`](../README.md#локальный-ai-опционально--всё-на-своём-пк).

### Проверьте готовность

Настройки → **🩺 Diagnostics** → **«Проверить всё»**. Покажет статус четырёх
подсистем: **AI**, **распознавание речи (STT)**, **микрофон**, **системный
звук**. Зелёным — готово к сессии. Это же — первое место, куда смотреть, если
что-то не работает.

## 4. Базовый рабочий процесс

1. **Запустите сессию** — нажмите на таймер/статус на баре (он станет
   «recording», зелёным). Захват системного звука включается автоматически;
   микрофон — тоже, если устройство выбрано.
2. **Идёт встреча.** В транскрипт попадают реплики; **авто-тайлы** сами
   открывают ответ, когда в речи собеседника появляется вопрос или ключевое
   слово (на ваш собственный голос детектор по умолчанию **не** реагирует).
3. **Нужен ответ прямо сейчас** — нажмите **F9** (ответ стримится в тайл).
4. **Ответ слабоват** — **Shift+F9** переспросит у более умной облачной модели
   (значок 🧠 на тайле), либо нажмите 🧠 на самом тайле.
5. **Продолжить диалог** — в тайле есть поле ввода: задайте уточняющий вопрос
   (текстом или 🎤 голосом), ответ придёт следующим сообщением в том же тайле.
6. **Скопировать ответ** — кнопка **📋** на тайле (для ветки копируется весь
   диалог, без служебного транскрипта).
7. **Остановите сессию** тем же кликом по таймеру. Если включён разбор —
   появится тайл **🎯 Debrief** с замечаниями по вашей речи.

## 5. Хоткеи (глобальные)

Все клавиши работают, **даже когда suflyor не в фокусе**.

| Клавиша | Действие |
|---|---|
| **F9** | Спросить AI сейчас — ответ стримится в тайл |
| **Shift+F9** | Спросить с эскалацией к умной облачной модели (глубже рассуждает, разово — не меняет провайдера по умолчанию; на тайле значок 🧠) |
| **F8** | Скриншот → Vision-AI: выделите область экрана мышью, **Esc** — отмена |
| **F3** | Переспросить последний вопрос с учётом свежего контекста |
| **F4** | Палитра базы знаний — открыть / закрыть |
| **F6** | Тайл вручную из последней реплики транскрипта |

**Запись голосом — удержанием** кнопок на баре (это не хоткей):
- **🎤 спросить** — удерживайте, говорите свой вопрос, отпустите → AI ответит;
- **🔊 захватить** — то же для звука собеседника.

> Полный список хоткеев есть и в приложении: **Настройки → ⌨ Hotkeys**.
> Клавиши глобальные — если какая-то не срабатывает, скорее всего её занял
> другой запущенный апп (suflyor пока не показывает об этом предупреждение).

## 6. Возможности по мелочам

- **🔊 / 🎤 на баре (тумблеры)** — какие источники звука захватывать.
- **✏ Написать** (чип на баре) — впечатать вопрос, когда говорить нельзя
  (шумно или нужна тишина). Окно ввода тоже прячется при скрытности.
- **🧠 эскалация** — Shift+F9 или кнопка на тайле: разовый переход на умную
  облачную модель; дальше диалог в этом тайле «липнет» к облаку.
- **🔄 экстренный перезапуск** (чип на баре) — перезапускает приложение,
  сохранив все настройки (включая скрытность). На случай, если что-то подвисло.
- **⚠ AI недоступен** — если модель отвалилась, бар честно об этом скажет
  (а не будет молча игнорировать F9 / авто-тайлы).
- **F8 → Vision** — распознаёт скриншот выделенной области и отвечает по нему
  (например, разобрать код или диаграмму с экрана).
- **F4 — база знаний** — ~1700 встроенных записей (глоссарий / команды /
  паттерны). Поиск → Enter/клик → тайл из записи, **без вызова AI ($0)**.
- **Профиль + контекст** — постоянный контекст, который добавляется к каждому
  запросу к AI (роль, стек, компания). Можно надиктовать голосом (🎤) и
  структурировать через AI (✨). Экспорт/импорт профиля — в Настройках.
- **Тайлы** — Markdown, перетаскивание, 📌 закрепить, развернуть, ✕ закрыть,
  «закрыть все». Прозрачность тела настраивается (Настройки → Скрытность).
- **$ лимит стоимости** — `max_session_cost_usd` (0 = выкл). Любое положительное
  число подсветит жёлтый чип при превышении (AI при этом **не** блокируется —
  только предупреждение). Локальные модели считаются как $0.

## 7. Скрытность (screen-share safety) — важно

Это ключевая фишка для собеседований по видеосвязи.

- Включение: **бар → чип скрытности** или **Настройки → 🛡 Скрытность**.
  По умолчанию **выключено** (overlay виден при шеринге).
- Что прячется: бар, все тайлы, палитра F4, окно Настроек и окно «Написать» —
  через Win32 `SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)`.
- Что это значит: окна **не попадают** в Teams / Meet / Zoom / OBS / Print
  Screen / стандартный захват экрана — вы их видите, собеседник нет.

**Проверьте заранее.** Включите скрытность и сделайте тестовый шеринг (или
Print Screen) до реального звонка — убедитесь, что бар и тайл не видны. Учтите:
аппаратный захват (камера на монитор, другой ПК) обойти невозможно — скрытность
работает только против программного захвата на этом же ПК.

## 8. Приватность и конфиг

- Конфиг: `%APPDATA%\overlay-mvp\config.json` (можно править вручную). Содержит
  ваши ключи — **не показывайте этот файл на шеринге**.
- Ключи (`groq_api_key`, `ai_bearer`) **никогда** не попадают в тайлы, логи или
  журнал сессий — даже в текстах ошибок (они намеренно обобщённые, чтобы адрес
  сервера/ключ не утёк на скриншот).
- **Локальный режим** — распознавание и LLM крутятся на вашем ПК, наружу не
  уходит ничего.
- Журнал сессий: `%APPDATA%\overlay-mvp\sessions\*.jsonl` (локально, для разбора
  после встречи). Лог приложения — `overlay-host.log` там же.

## 9. Решение проблем

| Симптом | Что делать |
|---|---|
| **SmartScreen не даёт запустить** | «Подробнее» → «Выполнить в любом случае». Бинарник не подписан — это ожидаемо. |
| **На баре «⚠ AI недоступен»** | Провайдер не отвечает. Настройки → AI → «Проверить подключение». Для локального — запущен ли llama-сервер. |
| **`stt · микрофон занят`** | Микрофон уже использует другой захват (PTT/диктовка). Подождите пару секунд и повторите. |
| **Нет транскрипта** | Идёт ли сессия? Выбран ли микрофон/устройство звука и задан ли ключ STT? Смотрите **🩺 Diagnostics → Проверить всё**. |
| **F9 «молчит»** | Проверьте, не «⚠ AI недоступен» ли на баре. Если нет — посмотрите `overlay-host.log`. |
| **Хоткей не срабатывает** | Глобальную клавишу мог занять другой апп. Закройте конкурента или проверьте в **Настройки → ⌨ Hotkeys**. |
| **Скрытность не прячет окно** | Включена ли она? Аппаратный захват (другой ПК/камера) обойти нельзя — работает только против программного захвата. |
| **Тайл улетел / не на том мониторе** | Тайл можно перетащить мышью. Бар при старте крепится к **основному** монитору. |
| **Ответ устарел / не гуглит** | Модель отвечает по своим знаниям + транскрипту; веб-поиска пока нет. Для свежего — формулируйте вопрос с фактами из встречи. |

Если ничего не помогло — `overlay-host.log` (`%APPDATA%\overlay-mvp\`) и
**🩺 Diagnostics** дают самую точную картину.
