# Запросы фич от тестера (2026-06-01)

Источник: обратная связь тестера по живому использованию (собеседования).
Статус: план/спека, код ещё не трогали. Оценки и точки интеграции заземлены
на актуальный код (slint-experiment @ v0.8.2).

Обозначения размера: **S** ≈ 0.5–1 ч · **M** ≈ 3–5 ч · **L** ≈ день+.

---

## Фича 1 — Кнопка «Копировать» на тайле ответа · **S**

**Сценарий.** Пользователь скриншотит задачу (Notion/Kaiten) → бот распознаёт и
выводит текст в тайле → пользователь хочет забрать текст и вставить дальше, но
кнопки нет, приходится выделять вручную.

**Что нужно.** Иконка 📋 рядом с каждым ответом бота; по клику — весь текст
ответа в буфер обмена; короткий фидбек («Скопировано» / смена иконки).

**Текущий код (точки касания).**
- `ui/tile.slint` — ряд «хрома» уже содержит кнопки 🔄 (`regen-touch`) и
  🧠 (`esc-touch`) как `Rectangle` + `TouchArea` + callback. 📋 встаёт туда же
  той же формой.
- Текст ответа: `OverlayBarBridge.conversations[convo_id]` хранит `messages`
  (Vec<ChatMessage>) и `rendered` (markdown-тред). Чистый источник для копии —
  последний `assistant`-месседж (raw markdown), без HTML/рендера.
- Буфер: app — только Windows. `arboard` (кроссплатформенный, `Clipboard::new()?
  .set_text(s)?`) либо напрямую `clipboard-win` (уже в дереве транзитивно через
  winit). Рекомендую `arboard` — простой API, на Windows под капотом тот же
  `clipboard-win`.

**Набросок реализации.**
1. `tile.slint`: добавить `copy-clicked()` callback + `copied: bool` property;
   новый `Rectangle`+`TouchArea` (📋) слева от 🔄; при `copied` — глиф меняется
   на фидбек-иконку.
2. `overlay_host.rs`: при создании тайла сохранить raw-текст ответа (или читать
   из `conversations` по `convo_id` в момент клика); в обработчике `copy-clicked`
   — `arboard::Clipboard::new()?.set_text(answer)`, затем выставить `copied=true`
   и сбросить через `Timer::single_shot(~1.5s)`.
3. `Cargo.toml`: `arboard = { version = "3", default-features = false }`.

**Нюансы / риски.**
- **Глиф фидбека.** В проекте уже обожглись: `✕`/`⟳` рендерятся «квадратом» на
  Slint+skia. Бэдж «скопировано» НЕ делать голым `✓` (U+2713, тоже text-глиф,
  риск квадрата) — взять эмодзи (✅) либо короткий текст «✓ Скопировано» эмодзи-
  шрифтом / тост. Проверить рендером (PrintWindow) до релиза.
- **Скрин-шеринг.** Копирование локальное, egress нет — безопасно. Сам тайл уже
  уважает стелс (WDA), новых окон фича не создаёт.
- **i18n.** Новые строки («Копировать», «Скопировано») — `@tr` + `ru.po`.
- **Что копировать.** Для типового Q→A — текст ответа (assistant). Для
  многоходового треда решить: только последний ответ или весь тред. Базово —
  последний ответ; полный тред можно дать вторым жестом позже.

**Гейты.** UI-изменение → полный цикл 1–5 (clippy/test/fmt/review/живой смоук +
PrintWindow на бейдж).

## Фича 2 — Текстовый режим «Написать» · **M**

**Сценарий.** Говорить нельзя (шум / людно / тишина). Пользователь жмёт
«Написать» → маленькое окно с полем ввода → печатает вопрос → ассистент отвечает
как обычно. Дополняет голосовой режим, не заменяет.

**Что нужно.** Кнопка вызова текстового ввода (на баре, рядом с микрофоном или
отдельно); маленькое окно с полем + кнопка отправки; ответ — в стандартном
тайле; не мешает основному интерфейсу.

**Текущий код (точки касания).**
- Уже есть наполовину: тайлы умеют ПРОДОЛЖАТЬ диалог печатно —
  `followup-text` ↔ `LineEdit`, `followup-submitted(string)`, `do-submit()`,
  `fire_followup_ask(...)`. То есть печатный ввод вопроса в существующий тайл
  работает.
- Чего НЕТ: входа «задать НОВЫЙ вопрос печатно с бара». Есть голосовые F9/PTT,
  F8 vision, ручной спавн (F6/`manual_spawn_tile`) и палитра F4 (поле ввода —
  но это поиск по KB, не вопрос ассистенту).
- Ask-машинерия для нового тайла: `fire_f9_ask` (берёт транскрипт) и ручной ask.
  Для печатного вопроса нужно подать ВВЕДЁННЫЙ текст как user-prompt (не
  транскрипт) и отрисовать ответ в новый тайл.

**Набросок реализации (вариант A — переиспользовать палитру/мини-окно).**
1. Бар: чип «✏ Написать» (или иконка рядом со стеком капче). Глиф — эмодзи
   (✏/⌨), не text-глиф.
2. Мини-окно ввода: либо новое маленькое Slint-окно (как `palette.slint`,
   frameless + один `LineEdit` + кнопка «Отправить»), либо расширить палитру
   режимом «вопрос ассистенту».
3. На submit: собрать запрос с введённым текстом как user-вопрос (по образцу
   `fire_followup_ask`, но в НОВЫЙ тайл/convo через ask+tile путь как у F9) →
   ответ стримится в стандартный тайл. Точные функции уточню при реализации
   (вероятно тонкая обёртка над общим ask-путём с явным текстом вопроса).

**Нюансы / риски.**
- **Стелс/скрин-шеринг (важно).** Окно ввода — НОВОЕ окно. Под стелсом обязано
  быть исключено из захвата (WDA) и не мигать — применить паттерн
  `present_window_stealth_aware` / park-before-show, как для палитры/Settings.
  Иначе на трансляции мелькнёт окно с печатаемым вопросом.
- **Фокус.** Окно ввода должно забирать фокус клавиатуры (как capture-overlay
  через `focus_window`), но мягко вернуть его после отправки/Esc, не ломая
  always-on-top бара.
- **i18n.** «Написать», «Отправить», плейсхолдер поля — `@tr` + `ru.po`.
- **Не мешать.** Закрытие по Esc/после отправки; не перехватывать глобальные
  хоткеи во время ввода.
- **Хоткей.** Желателен глобальный хоткей-дубль (как у F9/F8) для вызова без
  клика по бару — например, отдельная F-клавиша.

**Гейты.** Новое окно + ввод + ask-путь → полный цикл 1–5, обязательно живой
смоук под стелсом (Win32-проверка affinity=0x11 на окне ввода) и проверка, что
ответ корректно уходит в тайл.

## Сквозные заметки

- **i18n RU+EN.** Любая видимая строка — `@tr("English…")` в `.slint` +
  пара `msgid/msgstr` в `slint-replay.po`. Хардкод кириллицы без `@tr` = баг.
- **Скрин-шеринг — главный инвариант.** Любое НОВОЕ окно (ввод текста) обязано
  уважать стелс (WDA) и park-before-show. Буфер обмена — локальный, безопасен.
- **Глифы.** Только эмодзи для иконок (📋 ✏ ⌨ ✅); голые text-глифы (`✓ ✕ ⟳`)
  рендерятся квадратом на Slint+skia — проверять PrintWindow.
- **Методология.** Обе фичи трогают UI → полный цикл 1–5 + живой смоук перед
  релизом; без марафонов.

## Рекомендация по приоритету

1. **Копировать (S)** — быстрый винт, прямая боль тестера, минимальный риск.
   Можно выкатить отдельным патчем сразу.
2. **Текстовый режим (M)** — высокая ценность для сценария собеседования, но
   требует нового окна + аккуратной работы со стелсом/фокусом. Отдельный заход.
3. **Фича 3** — после уточнения.
