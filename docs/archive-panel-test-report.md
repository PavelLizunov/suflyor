# Архив-панель — отчёт живой проверки (заполни и верни мне)

**Как заполнять:** после каждого `→` напиши коротко:
`OK` · `БАГ: <что не так>` · `? <вопрос/непонятно>` · `СКИП <почему не проверил>`.
Скриншоты приветствуются (особенно на БАГ) — приложи или опиши.

**Сборка:** master `cb708e5` (все 4 ТЗ), debug-запуск `cargo run --bin overlay-host`.
**Дата проверки:** _________  **Тема:** _________ (Light Frost / тёмная)

Открыть архив — **F7**. Стенограмма/плеер — **📄** в строке сессии **с записью**.

---

## 1. ПЛЕЕР (ТЗ2б) — ПРОВЕРЯТЬ НА СЛУХ (я это не мог вообще)

**1.1** Клик по строке → микс (микрофон+система) играет именно С МОМЕНТА этой строки.
→

**1.2** Кнопка ▶/⏸ — play/pause работает, иконка меняется.
→

**1.3** Клик по другой строке во время игры → перескок туда, играет дальше.
→

**1.4** Seek-bar: клик в точку полосы → перескок туда (на слух); заполнение ползёт при игре.
→

**1.5** Время «mm:ss / mm:ss» идёт и совпадает с реальной позицией.
→

**1.6** Подсветка активной строки двигается за воспроизведением.
→

**1.7** Дослушать до конца → следующий ▶ играет С НАЧАЛА (не с последней точки).
→

**1.8** Закрыть окно во время игры → звук сразу прекращается.
→

**1.9** Сессия без записи → плеера нет, написано «Аудио не сохранено».
→

**1.10** Звук не двоится / не трещит, без лишних задержек.
→

---

## 2. СТЕНОГРАММА (ТЗ1) — окно по 📄

**2.1** Заголовок окна «Стенограмма» (по-русски).
→

**2.2** Строки: `[mm:ss]` · Система/Микрофон · текст; таймкоды/метки совпадают с обычным видом сессии.
→

**2.3** Длинные реплики переносятся по словам, не обрезаются.
→

**2.4** Старая сессия без строк → «Транскрипт не сохранён», без падения, без пустой колонки времени.
→

**2.5** «Копировать всё» → вставь в блокнот: формат «Спикер: текст» по строкам; кнопка мигнула «Скопировано ✓».
→

**2.6** Галка «с таймкодами [mm:ss]» ВКЛ → в копии префикс `[mm:ss] `.
→

**2.7** Чекбоксы строк + «выделить все»: отметь несколько → «Копировать выбранное» → в буфере только выбранные, в хронологии.
→

**2.8** Esc закрывает окно; перетаскивание за заголовок двигает окно.
→

---

## 3. УДАЛЕНИЕ (ТЗ2а) — 🗑 в строке (тестируй на НЕНУЖНОЙ сессии — необратимо!)

**3.1** 🗑 → диалог подтверждения; Отмена → ничего не удалилось.
→

**3.2** Подтверждение → сессия исчезла из списка сразу и не находится поиском.
→

**3.3** На диске стёрто: `.jsonl`, `recordings/<id>/`, `conspects/<id>.json`, строка каталога (`%APPDATA%\suflyor\`).
→

**3.4** Активную (текущую идущую) сессию удалить нельзя.
→

**3.5** Повторное удаление уже удалённой — без ошибки/мусора.
→

---

## 4. РЕТРО-РЕЗЮМЕ (ТЗ3) — ↻ Summary в строке

**4.1** Сессия без резюме и без аудио: ↻ активна → строит резюме из транскрипта; прогресс + стоимость; язык = response_language.
→

**4.2** Нет данных → кнопка неактивна.
→

**4.3** Есть резюме → «Пересоздать» только с подтверждением; ошибка → «Повторить», сессия не портится.
→
