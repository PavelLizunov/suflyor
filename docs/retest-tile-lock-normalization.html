Суфлёр — приёмка: замок тайлов + нормализация памяти

На УСТАНОВЛЕННОМ билде (после setup.exe /S и запуска установленного ярлыка — не из target/release). Отметь каждый пункт, добавь комментарий, «Экспорт результатов» → пришли мне текст.

Что проверяем (2 фичи)

  • 🔒 Замок тайлов — кнопка-замок на баре: глушит всплывание авто-тайлов (например, при записи игры), дрожит пока активна.
  • 🧠 Нормализация памяти (M1-b-2) — ⭐ на строке стенограммы: факт сохраняется сразу, затем ИИ фоном подчищает его в аккуратный факт (числа/IP/имена — дословно, иначе переписка отвергается).

🔒 Замок тайлов

Кнопка-замок на баре
На баре в ряду быстрых действий (рядом с 🔥 «агрессивный авто-тайл») появился значок замка — векторный, в стиле остальных иконок.
Клик → подсветка + дрожь
Клик по замку → он подсвечивается (янтарный) и потрясывается (мелко дрожит). Повторный клик → выключается, замок замирает ровно (не остаётся наклонённым).
Авто-тайлы заглушены / вернулись
Замок активен → говоришь, но авто-тайлы не вылетают. Выключил замок → авто-тайлы снова появляются на каждую реплику (как раньше).
Ручное всё ещё работает
При ВКЛЮЧЁННОМ замке: F9/«Написать»/F6, «Сводка», ошибки — тайлы всё равно появляются (замок глушит ТОЛЬКО авто-тайлы, не осознанные запросы).
Один переключатель с Настройками
Замок на баре и Настройки → «Режим прослушивания» («не показывать тайлы») — это ОДНО и то же. Переключил на баре → открыл Настройки → галка в том же состоянии (и наоборот).
Сохраняется после перезапуска
Включил замок → закрыл и снова открыл приложение → замок всё ещё активен и дрожит (сразу понятно, почему тайлов нет).

🧠 Нормализация факта из стенограммы

Мгновенно, затем подчищается
Отметь ⭐ ОДНУ строку стенограммы (сырой STT, с «эканьем»/повторами) → «В память». В Настройки→Память факт появляется сразу; через миг (после ответа ИИ) текст подчищается в аккуратный факт.
Идентификаторы — дословно
Если в реплике были число / IP / подсеть / порт / имя (напр. 10.255.28.116, z14-4443) — в подчищенном факте они ТЕ ЖЕ, ни одна цифра/буква не изменилась. (Если ИИ что-то изменил в идентификаторе — переписка отвергается, остаётся исходный текст.)
Смысл/отрицание не переврано
«не», «нельзя», «без» не перевернулись; факт означает то же, что сказано. Ничего выдуманного не добавилось.
Без ИИ — тоже работает
Если ИИ выключен / локальная модель не настроена — факт всё равно сохраняется (просто вычищенный эвристикой, без переписки). Ничего не теряется, ⭐ не зависает.
Остальное ИИ НЕ трогает
⭐ на БЛОКЕ тайла, «свой факт» в Настройках, склейка нескольких ⭐ в одну запись, выделенный фрагмент — сохраняются КАК ЕСТЬ (ИИ их не переписывает — только сырую одиночную ⭐-строку стенограммы).
Старое не сломалось
v0.27-поведение цело: несколько ⭐ → ОДНА склеенная запись; редактор факта перед сохранением; копирование; «Продолжить диалог».
