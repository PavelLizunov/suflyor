🧠 Баг 1 — память отвечает по фактам

A1 (главное)Точный вопрос про сущность из памяти — ответ ИЗ памяти
Убедись, что в «Одобренной памяти» есть факт про людей (Тимур Писчаскин / Михаил Голубцов / Владислав Кощеев). Спроси через ✏ Написать: кто такой Владислав Кощеев? → ответ содержит факт из памяти (роль/контекст), НЕ «нет информации».
＋ перетащи/вставь скриншот
A2 (главное)Сокращение и опечатка тоже находят факт
Спроси: кто такой Влад Кощеев? (сокращение) и кто такой Тимур Писчанкин? (опечатка в фамилии) → в обоих случаях ответ из памяти. Это ровно те кейсы, что падали.
＋ перетащи/вставь скриншот
Старый факт (не из последних 8) тоже находится
Если одобренных фактов больше 8 — спроси про один из самых старых. Раньше он гарантированно выпадал; теперь релевантность его поднимает. (Если фактов ≤8 — пропусти.)
Обычные вопросы не сломались
Пару обычных вопросов без связи с памятью (голосом/✏) — ответы адекватные, память не «лезет» не по делу.

📸 Фича 2 — скрин-тайл в умное облако

B1В «⋯» скрин-тайла есть «Smart cloud»
F8 (скрин) → в тайле ответа открой меню «⋯» → есть пункт Smart cloud (как у текстовых). Примечание: пункт показывается, когда vision-ответ шёл через локальную модель и настроен облачный ключ; в OCR-режиме (Ctrl+F8) его нет — это правильно.
＋ перетащи/вставь скриншот
B2Перегенерация шлёт ТОТ ЖЕ скрин в облако
Нажми «Smart cloud» → тайл помечается «cloud (escalated)», ответ перегенерируется по тому же скриншоту (заметно глубже/качественнее). Скрин заново НЕ снимается. Голосовой follow-up после эскалации снова идёт через локальный vision — эскалация одноразовая, так задумано.
＋ перетащи/вставь скриншот

🖱 Фича 3 — перемещаемая плашка ввода

C1Плашка таскается мышью
Открой «✏ Написать» → схвати за заголовок («Напишите вопрос ассистенту») → перетащи в любое место, включая второй монитор. Курсор над заголовком — «рука». Крестик и поле ввода работают как раньше.
＋ перетащи/вставь скриншот
C2Позиция запоминается
Перетащи плашку → закрой (Esc или отправь вопрос) → открой снова → она там же. Перезапусти приложение → всё ещё там же.
Ввод/фокус/Esc не пострадали
После перетаскивания: печать сразу идёт в поле, Enter отправляет, Esc закрывает, Ctrl+C/V на русской раскладке работают. (Drag не должен красть фокус.)
